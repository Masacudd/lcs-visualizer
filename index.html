Key Concepts
  • Dynamic Programming stores results of subproblems.
  • The LCS table grows row by row and column by column.
  • Traceback reconstructs the subsequence.
Core Idea

The Longest Common Subsequence (LCS) problem finds the longest sequence of characters that appears in both strings in the same order, but not necessarily consecutively. The algorithm uses Dynamic Programming to store partial solutions and avoid recomputing the same subproblems.

Each cell in the table represents the length of the LCS for prefixes of the two strings. By filling the table step-by-step we compute the final solution.


How It Works (Step by Step)
  1. Create table L with size (m+1) × (n+1)
  2. Initialize first row and column with 0
  3. Fill the table row by row
  4. Compare characters X[i-1] and Y[j-1]
  5. If equal → L[i][j] = L[i-1][j-1] + 1
  6. If different → L[i][j] = max(L[i-1][j], L[i][j-1])
  7. Continue until the table is filled
  8. The value L[m][n] is the length of the LCS
  9. Traceback from bottom-right to reconstruct the subsequence

Complexity

Worst / Average: O(m × n) | Best (very small common prefixes): O(m × n) | Space: O(m × n) | Stable: ✅ Yes


When to Use
  • DNA sequence comparison
  • Version control diff tools
  • Plagiarism detection
  • Text similarity analysis
def LCS(X, Y):
    m = len(X)
    n = len(Y)

    # Step 1: Create DP table
    L = [[0] * (n + 1) for _ in range(m + 1)]

    # Step 2: Fill the table
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i - 1] == Y[j - 1]:
                L[i][j] = L[i - 1][j - 1] + 1
            else:
                L[i][j] = max(L[i - 1][j], L[i][j - 1])

    # Step 3: Traceback to get LCS string
    i = m
    j = n
    lcs = []

    while i > 0 and j > 0:
        if X[i - 1] == Y[j - 1]:
            lcs.append(X[i - 1])
            i -= 1
            j -= 1
        elif L[i - 1][j] > L[i][j - 1]:
            i -= 1
        else:
            j -= 1

    lcs.reverse()

    return L[m][n], "".join(lcs)
What You Are Seeing
  • Table: Represents LCS values for prefixes of X and Y.
  • Rows: Characters of string X
  • Columns: Characters of string Y

Cell Colors
  • Green: Characters match (diagonal move)
  • Orange: No match (take max of top/left)
  • Dark cells: Normal computed values
  • Red border: Traceback path (building LCS)

Arrows Meaning
  • ↖ (Diagonal): Match found → extend subsequence
  • ↑ (Up): Take value from above
  • ← (Left): Take value from left

How to Read the Visualization
  1. Start from top-left and fill the table step by step
  2. Each cell compares characters from X and Y
  3. Green cells indicate matches
  4. Final answer is in the bottom-right cell
  5. Traceback shows how the LCS string is formed

  • Brute Force (O(2ⁿ)): grows extremely fast and becomes impractical.

As the input size increases, the gap between the two approaches becomes very large, showing why Dynamic Programming is preferred for solving the LCS problem.

Why Brute Force is Inefficient
Brute force generates all possible subsequences of a string (2ⁿ possibilities). Each subsequence must then be checked against the second string, which makes the algorithm extremely slow for large inputs.
